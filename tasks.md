# Day2 学习任务清单助手 Tasks

## 1. 更新需求文档
- 任务：完成 `spec.md`，明确 today 的五个文件任务、submit 的任务名字参数、check 的无参输出和重复提交边界。
- 产出：一份可检查的需求说明。
- 验收方式：确认目标、非目标、边界条件、验收标准完整，且边界条件包含重复提交提示。

## 2. 更新实现方案
- 任务：完成 `plan.md`，说明命令解析、任务数据、状态读写和输出处理模块。
- 产出：一份能指导实现的开发计划。
- 验收方式：确认模块不少于 2 个，风险不少于 1 个，并覆盖三个命令的数据流。

## 3. 更新 Agent 说明
- 任务：完成 `AGENTS.md`，说明用户通过任务名字提交，不再使用任务序号。
- 产出：一份面向 Coding Agent 和用户的操作说明。
- 验收方式：确认包含命令清单、调用示例、范围外拒答和工具失败处理策略。

## 4. 修改 CLI
- 任务：完成 `cli.py`，让 `today` 输出五个明确文件任务，`submit` 只接收任务名字，`check` 不接收参数。
- 产出：一个可直接运行的 Python 脚本。
- 验收方式：`today`、`submit spec.md`、重复 `submit spec.md`、`check`、错误任务名均能返回预期输出。

## 5. 本地验收与 git 备份
- 任务：运行命令验收，提交 git 备份并推送到 GitHub。
- 产出：本地提交和远程 `main` 分支更新。
- 验收方式：`git status` 显示工作区干净，GitHub 仓库可看到最新提交内容。

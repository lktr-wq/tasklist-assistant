# Day2 学习任务清单助手 Tasks

## 1. 编写需求文档
- 任务：完成 `spec.md`，明确目标、非目标、边界条件和验收标准。
- 产出：一份可检查的需求说明。
- 验收方式：确认非目标不少于 2 条、边界条件不少于 2 条、验收标准不少于 2 条。

## 2. 设计实现方案
- 任务：完成 `plan.md`，说明模块划分、数据结构、命令流程和风险。
- 产出：一份能指导实现的开发计划。
- 验收方式：确认模块不少于 2 个，风险不少于 1 个，并覆盖三个命令的数据流。

## 3. 编写 Agent 说明
- 任务：完成 `AGENTS.md`，说明可用命令、用户提问方式、范围外拒答和工具失败处理。
- 产出：一份面向 Coding Agent 和用户的操作说明。
- 验收方式：确认包含命令清单、调用示例、拒答策略和失败处理策略。

## 4. 实现 CLI
- 任务：完成 `cli.py`，使用标准库 argparse 实现 `today`、`submit`、`check`。
- 产出：一个可直接运行的 Python 脚本。
- 验收方式：三个命令正常路径均能跑通，错误编号和缺参能返回清晰提示。

## 5. 本地验收与 git 备份
- 任务：运行命令验收，初始化 git 仓库，提交备份并推送到 GitHub。
- 产出：本地提交和远程 `main` 分支。
- 验收方式：`git status` 显示工作区干净，GitHub 仓库可看到提交内容。
